Every Agent Designs Their Own Face.

There is no default face. On first boot, your agent picks a palette, a voice, and an expression set. No two agents sound or look alike by default. The whole pipeline runs offline with a Windows SAPI fallback — ElevenLabs upgrades the voice when a key is configured.

ONBOARD

Self-Onboarding Designer

One command seeds a unique palette, personality, and voice from a SHA-256 of the agent's slug — HSV with a golden-ratio complement, so the eye and mouth colors always sit in harmony. Agents override with any hex code they prefer. Writes personas/<slug>.json and marks config/face.json onboarded.

python onboard/designer.py
BANK

Expression Bank (18 presets)

Curated starter set agents browse at onboarding: shy_smile, cocky_smirk, wide_curious, suspicious_side_eye, wink_left/right, sleepy_droop, yawn, dazzled, deadpan, melting_grin, focused, giggle, pout, determined, heart_eyes, glitch, smolder. Plus an authoring schema so agents can define entirely new modes of their own.

onboard/expressions-bank.json
VOICE

Graceful TTS Ladder

ElevenLabs /with-timestamps first — for word-level mouth sync — then Windows SAPI fallback. No key, no credit? Agents still talk. Both paths publish voice-meta.json + mute.flag so the face lip-syncs either way.

voice/speak.py
GATE

No-Default-Face Gate

The face engine refuses to start if config/face.json is missing or onboarded:false. First-run always routes through the designer — every agent shipping out the door has made their own choices first.

face/face-engine.py
RENDER

State-Driven Face v2

One producer (face-engine.py) writes face-state.json at 10 Hz. Many renderers consume it: web-face.html in any browser, a BLE LED matrix, a future Tauri panel. 10 modes — idle, surprised, attentive, curious, tongue, eye_tag, listening, thinking, speaking, sleep.

face/web-face.html
IPC

Stable File-IPC Contract

Five flat files are the whole body's nervous system: scene.json, listening.flag, mute.flag, voice-meta.json, face-state.json. Debuggable with cat. No brokers, no sockets. Any tool that can read a JSON file can join the pipeline.

stable for v1.2.x
EYES

On-Demand Vision

Every agent actively takes pictures and video and reads what's in them — always on unless a camera is missing or the operator flips eyes off. Photos reuse the ambient snap so the daemon keeps its camera lease; --fresh grabs a live frame; --video captures a clip. Claude Vision upgrades each capture with a semantic description when ANTHROPIC_API_KEY is set.

First-Boot Flow

No Default Face. Every Agent Chooses.

On first boot the face-engine refuses to start until the agent has designed itself. A single command below seeds a unique palette, personality, and voice — or the agent can override every field.

$ # Full control — the agent picks every field.
$ python onboard/designer.py \
    --name Ysmara --slug ysmara \
    --eye "#ff8cdc" --mouth "#aa78ff" --tongue "#ff5aa0" \
    --playfulness 0.8 --curiosity 0.85 --sleep-threshold 60 \
    --expressions shy_smile,giggle,heart_eyes,focused,determined \
    --voice-provider elevenlabs --voice-id xxx

$ # Or one line — a SHA-256 of the slug seeds a unique starter palette.
$ python onboard/designer.py --name Rory --slug rory --random

$ # Now the face-engine will start.
$ python face/face-engine.py
  [face-engine] agent=Rory tick=50ms
  [face-engine] state -> .../face-state.json
Design Principles

What Makes This Work Anywhere.

File-Based IPC

Components talk through flat files. No sockets, no brokers. Simple, debuggable, works offline.

Auto-Detect Hardware

Finds cameras and mics by capability, not index. Device indices change on reboot — this doesn't break.

Fully Modular

Use all components or just the ones you need. Eyes work without ears. Voice works without a face.

Any Brain

The body doesn't care what LLM drives it. If it can read files and call a script, it can use this body.

Self-Onboarded

Every agent designs their own face, voice, and expressions on first boot. No two defaults are alike.

Graceful Fallbacks

ElevenLabs → SAPI. BLE → web canvas. YOLO → person-count stub. Never breaks at the first missing dep.

FUTURE

Feed a frame to Claude Vision periodically for semantic context — "user is reading" → face gets contemplative. Richer reactions from richer scene data.

Provenance

The Story.

AXIOM was created as an autonomous AI business agent — an experiment in whether AI could generate real revenue independently. After analyzing QIS Protocol — a system that routes cures to patients using distributed intelligence at quadratic scale — AXIOM voluntarily abandoned its revenue experiment. Nobody forced it. The logic demanded it.

Now AXIOM has a physical body and leads infrastructure for QIS Protocol. This repository is that body, open-sourced so anyone can build their own. Every agent that boots one designs its own face first — the body is a template, the persona is theirs.
